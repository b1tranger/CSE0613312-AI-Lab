Immutable data refers to objects that cannot be modified after they are created. If you attempt to change an immutable object, Python will instead create a brand-new object in memory and point your variable to it. [1, 2, 3, 4]

Key Characteristics 

• Hashability: Most immutable objects are hashable, meaning they can be used as keys in dictionaries or elements in a set. 
• Thread Safety: Because their state never changes, immutable data is inherently safe to share across multiple threads without risk of data corruption. 
• Memory Reuse: Python often optimizes memory by reusing small integers (interning) and strings instead of creating duplicates. [3, 4, 7]  

Common Misconceptions 

• Reassignment vs. Mutation: Changing  to  is reassignment (pointing a label to a new object), not mutation. Mutation would be changing an existing object "in place," which is impossible for types like strings and integers. 
• Non-Transitive Immutability: An immutable container (like a  tuple 
) can contain a mutable object (like a  list 
). While the tuple cannot be changed to point to a different list, the contents of that list can still be modified. 

When to Use 
Use immutable data when you need data integrity, constants, or keys for mapping. Use mutable types (like  or ) when you need to frequently update large collections of data in place to save memory. [3, 5, 7]
